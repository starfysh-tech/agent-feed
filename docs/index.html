The visibility gap

Code diffs show what changed. Not why.

Hidden decisions

Agents choose between approaches, pick defaults, apply patterns. None of this appears in the diff. You review the output without seeing the reasoning.

Unvetted dependencies

A new library gets pulled in because the agent thought it was the right call. You find out during deploy — or worse, in production.

Invisible workarounds

“For now I’ll use string concatenation” — temporary code that becomes permanent because nobody saw it happen.

How it works

Zero friction. Structured visibility.

Agent Feed is a transparent proxy. Set one environment variable — your agents route through it without any changes to their setup.

Classification

Nine types of agent behavior, classified automatically.

Each response is analyzed and tagged. These aren’t vague sentiments — they’re specific, reviewable items with confidence scores.

decision
A choice between alternatives
assumption
Something assumed without verification
architecture
A structural or design choice
pattern
A design pattern or convention applied
dependency
A library or service introduced
tradeoff
An explicit this-over-that choice
constraint
A hard limit shaping the approach
workaround
A temporary or non-ideal solution
risk
Something potentially problematic

The interface

Built for post-session review.

Walk through each flagged item in order. Accept, flag for changes, or mark false positives. Every review action trains the classifier.

Spot patterns across sessions. Rising workaround rates. Recurring wrong assumptions. Measure whether prompts are improving over time.

Every agent session captured with flag counts, agent type, and git context. Filter by agent, date, or repository.

Use cases

Visibility at every stage.

Pre-merge audit

Before reviewing a PR, pull up the session. Know what was decided — what alternatives were considered, what was assumed — before reading the code.

Team oversight

When developers use agents, you lose visibility into what the agent decided on their behalf. Agent Feed creates the audit trail without adding documentation burden.

Prompt improvement

Trends reveal which prompts produce workarounds, which tasks generate wrong assumptions. Fix the inputs, not the outputs.

Classifier calibration

Built-in eval framework measures precision and recall per flag type. Your review actions become ground truth automatically.

Get started

One install. One command. One env var.

terminal 
git clone https://github.com/starfysh-tech/agent-feed
cd agent-feed && npm install && npm link
agent-feed start
# That's it. Your agents now route through the proxy.

Works with Claude Code, Codex, and Gemini. Requires Node.js 20+. All data stays local in ~/.agent-feed/.

Under the hood

Simple architecture. No surprises.

Storage
SQLite with WAL mode, local at ~/.agent-feed/
Classification
Haiku by default. Supports Ollama and LM Studio for fully local inference.
Security
API keys scrubbed before storage. Auth headers never written to disk.
Eval
Built-in precision/recall/F1 metrics per flag type. CLI-driven.
Telemetry
Native OpenTelemetry receiver alongside the proxy. Captures tool decisions, hooks, and MCP server lifecycle that proxies can't see. Claude Code and Gemini CLI emit when telemetry env vars are set; Codex via config.
Restart safety
Atomic schema migrations. agent-feed start verifies the daemon is actually serving requests via /api/health before reporting success; on failure, env vars are cleared so coding agents fall through to direct upstream rather than hanging.
Architecture
ESM Node.js. No build step. No framework. 204 tests.
