## Design Context

### Users
Solo developer reviewing AI coding agent decisions across multiple agents (Claude Code, Gemini, Codex) and repos. Used intermittently — comes back after days/weeks, needs to quickly scan what agents decided, triage the obvious, and drill into the few that need attention. Viewing context: desktop, dark room, late sessions or between coding tasks.

### Brand Personality
Calm, focused, unhurried. Like a well-organized research notebook that also has the review mechanics of a code review tool. Not a monitoring dashboard — a judgment tool.

### Aesthetic Direction
- **Theme**: Dark. Developer context, late-session usage, code-adjacent tool.
- **Visual tone**: Between GitHub PR review (inline, expandable, threaded) and Grafana (dense, data-visible). Calm over flashy. Information hierarchy over decoration.
- **Anti-references**: Generic admin dashboards. Colorful card grids. Anything that looks like "AI built this."
- **Typography**: Monospace for data/labels, proportional for reading. High contrast between levels.

### Design Principles
1. **Scan first, read second** — the default view should let you triage 20 flags in 30 seconds. Detail is revealed, not imposed.
2. **Reviewed items recede** — once you've made a judgment, the flag should visually step back so unreviewed items dominate.
3. **Confidence drives attention** — lower confidence = needs more scrutiny. Visual weight should follow.
4. **Notes are secondary** — most flags get a quick accept/reject. Note fields appear only when needed.
5. **No ceremony** — no transitions for their own sake, no loading spinners where data is instant, no modals.
